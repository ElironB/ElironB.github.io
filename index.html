Technical footprint

The public product and the engineering story live together.

The Sellable codebase combines a web chat experience and a phone agent workflow. The current application is built in Next.js 16 with a PostgreSQL layer on Supabase, plus integrations for VAPI, Telnyx, Nylas, and payment infrastructure through Polar.

That combination matters because the value is operational, not decorative. Sellable is designed to sit in the middle of listing response: one product that can answer, qualify, message, schedule, and surface context back to the agent.

Web chat agent

Streaming AI workflows for in-app guidance and agent tooling.

Voice agent

Webhook-based phone automation for inbound listing coverage.

Calendar and messaging

Nylas and Telnyx support scheduling and follow-up loops.

Database and billing

Supabase, Drizzle, and Polar keep account and plan logic grounded.

FAQ

Short answers for common questions.

What is Sellable?

Sellable is an AI listing operations platform for real estate agents. It helps agents answer calls and texts, qualify buyers, send follow-up messages, and coordinate showings with less manual work.

Who is it built for right now?

The current wedge is solo agents and small teams managing active listings. The product is intentionally focused on listing response operations first.

Is this the live app?

No. This is the public GitHub Pages profile for the project. The live application experience lives at sellabl.app/agents.

Where can I learn more?

You can visit the main site at sellabl.app or review the public Pages source on GitHub.
